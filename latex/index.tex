\hypertarget{index_autotoc_md10}{}\doxysubsection{How to get vcpkg for dependency management?}\label{index_autotoc_md10}
Note; This is a global configuration step, and thus only needs to be performed once, and once only, on the entire operating system.

The following references the steps provider by in the official documentation (no affiliations) at\+: \href{https://vcpkg.io/en/getting-started.html}{\texttt{ https\+://vcpkg.\+io/en/getting-\/started.\+html}}

Step 1\+: Clone (or Fork) the vcpkg repo

\begin{DoxyVerb}git clone https://github.com/Microsoft/vcpkg.git
\end{DoxyVerb}
 

Step 2\+: Run the bootstrap script to build vcpkg

\begin{DoxyVerb}.\vcpkg\bootstrap-vcpkg.bat
\end{DoxyVerb}
 

Step 3\+: Run the vcpkg IDE Integration installer command

\begin{DoxyVerb}.\vcpkg\vcpkg integrate install
\end{DoxyVerb}
 

Step 4\+: Create a new Environment variable called VCPKG\+\_\+\+ROOT pointing to the root folder of your local vcpkg repo

\begin{DoxyVerb}For Windows OS:

1. Hit "search" button in the Windows taskbar, and type "env...";

2. Choose "Edit the system environment variables" from Control Panel;

3. Click the "Environment Variables..." button;

4. Under the "System Variables" entries, click "New...";

5. For "Variable name", enter "VCPKG_ROOT" (all caps, no quotation marks!)

6. For "Variable value", enter the absolute path to your local vcpkg repo's root folder, i.e., C:/path/to/vcpkg

Example:

Edit system variable

Variable name: VCPKG_ROOT
Variable value: C:/Code/vcpkg

Click "ok" and restart your Powershell.
\end{DoxyVerb}
 

Once complete, vcpkg should now be cloned into your chosen directory, with an environment variable pointing to that directory. The \char`\"{}bootstrap\char`\"{} and \char`\"{}integrate\char`\"{} commands in steps 2 and 3 have set up the host machine for vcpkg dependency management. A couple of special files in our build take care of all the rest (keep reading for more).

You now have everything you need to begin your CMake project. You may go ahead and open the cloned repo\textquotesingle{}s root folder (containing the upper-\/most \char`\"{}\+CMake\+Lists.\+txt\char`\"{} file) in your IDE of choice (i.\+e., Visual Studio 2022 or VSCode!), where the project will begin configuring several Cache files (and downloading/resolving package dependencies via vcpkg), generating the binary build files in the process.\hypertarget{index_autotoc_md11}{}\doxysubsection{$<$em$>$\+Tip 1$<$/em$>$\+:}\label{index_autotoc_md11}
